Ajax向服务端提交Map数据

发表于 2017-04-02 | 阅读次数

SpringMVC Controller代码

服务端所需数据FriendlyLink

Ajax写法(使用了jQuery库)

分析:

SpringMVC解析规则 : 根据类的属性字段名,如果有相同名称且数据类型相吻合的数据提交到Controller,就将其解析为相应的类。 如本例中的FriendlyLink类中只有一个名为friendlyLink的字段,就需要客户端提交过来一个相应名字的数据。
我们看Ajax的data : {friendlyLink:{"谷歌":"google.com","百度":"baidu.com"}},这是一个标准的JSON写法， Key:friendlyLink对应服务端所需的FriendlyLink类(注意不是看类名,而是类的属性字段!!!)。 Value:{“谷歌”:”google.com”,”百度”:”baidu.com”},对应的是friendlyLink所对应的数据类型Map, Map对应的JSON也是Key:Value对。对照着data数据的Value值, 相信大家已经知道怎么写Map相应的JSON了吧。

注意:

SpringMVC不能解析List<…>、Map<…>的数据,必须将其封装起来才可以!
