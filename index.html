定义一个简单实体

MongoDB 是 NoSQL文档数据库。在这个例子里，你要存储Customer这个对象。
src/main/java/hello/Customer.java

这里你创建了一个Customer类，这个类有三个属性，id，firstName，和lastName。id这个属性通常是MongoDB内部使用的。你同样要有一个构造函数用来创建新的实例。

在本向导中，为了更简洁去除了典型的Get/Set方法。

id与MongoDB的标准字段id相匹配，所以在Spring Data MongoDB中不需要特殊的标签去注解它。
另外两个属性，firstName和lastName，都不需要注解。他们最终会被映射成和他们变量名一样的字段。
toString()方法会用一种舒服的方式把一个Customer的信息打印出来。

MongoDB存储数据在集合里。Spring Data MongoDB会映射Customer类到一个名为customer的集合。如果你想改变集合的名字，可以使用Spring Data MongoDB的注解@Document来给一个类定义一个别的集合名。

创建简单查询

Spring Data MongoDB 关注于向MongoDB存储数据。它同样继承了Spring Data Commons项目中函数，比如查询的能力。本质上，你不需要学习MongoDB的查询语言；你只需要少量的方法，其他的查询会帮你做。
观察它是如何工作的，创建一个用来查询Customer文档的Repository接口。
src/main/java/hello/CustomerRepository.java

CustomerRepository 继承MongoRepository接口并且指定了运行时的类型：Customer和String。简单易用，这个接口自带了很多操作，包括基本的CRUD操作(create-read-update-delete)。
你可以声明一些其他的你需要的方法，只需要声明它们的方法。在这里，添加了findByFirstName，本质上这段代码等同于从文档中查找一个匹配firstName 的 Customer。
同样有一个findByLastName的方法查询一个符合条件的查询。
在一个典型的Java应用程序中，你需要写一个类实现CustomerRepository并且自己实现这些方法。什么让Spring Data MongoDB变得如此有用？事实上，你不需要写任何的方法实现。Spring Data MongoDB会在你运行程序时自动创建这些方法。
让我们把它跑起来看看它怎么运行的！

创建一个Application类

这里你创建了一个Application类和所有组件。
src/main/java/hello/Application.java

@SpringBootApplication是一个易用的注解，它包括以下内容：
@Configuration标签标注这个类作为应用上下文的资源。
@EnableAutoConfiguration告诉Spring Boot在启动时加载在classpath设置资源，其他bean，和可用的属性设置。
一般情况下你会给Spring MVC应用添加一个@EnableWebMvc的注解，但是Spring Boot会自动添加这个注解如果它在classpath下看到spring-webmvc。这个注解标志着应用是一个web应用，并且会像配置了DispatcherServlet一样启动关键行为。
@ComponentScan告诉Spring去在hello包下查找其他组件，配置，和Service，允许它查找Controller。
main()方法用了Spring Boot的SpringApplication.run()方法去加载一个应用程序。不知道你有没有注意到我们并没有写一行的XML？也没有其他的web.xml文件。这个应用是用纯Java代码编写的，并且你不需要配置其他的东西。

默认情况下，标注了@EnableMongoRepositories的类会扫描当前包下的任何继承了Spring Data的repository接口的接口。如果你的项目中包含多个子项目，并且程序不能正确找到你想要使用的repository，那么使用basePackageClasses=MyRepository.class会安全的告诉Spring Data MongoDB去扫描不同根路径下的包来找到相应的repository。

Spring Data MongoDB 使用MongoTemplate执行你的所有find*`方法。你可以使用这个模板来定义一些符合的查询，不过本向导里不包含这些。 Application包含一个main()方法并且自动装载了一个CustomerRepository的实例：Spring Data MongoDB动态的创建一个代理并注入。我们通过一些简单的Test来使用CustomerRepository。首先，它存储了一些Customer对象，通过sava()方法和设置一些必要的参数。接下来，调用findAll()方法从数据库中查找符合条件的Customer对象。然后调用findByFirstName()方法查找符合条件的单个Customer对象。最后，调用findByLastName()**方法查找姓氏为’Winstone’的Customer。

Spring Boot 默认会连接本地的MongoDB。如果要连接远程的MongoDB，请自己查找相关资料。

机器学习框架Darknet的Windows版

发表于 2017-11-16 | 阅读次数

这段时间将运行在Linux平台的机器学习项目Darknet迁移到了Windows平台。期间大问题没有小问题不断，经过几天的努力差不多已经完成整个项目的迁移。在此记录一下遇到的比较头疼的问题。

想要在Windows平台学习Darknet可以参考Darknet

提示：显卡选择很重要，支持CUDNN的Nvidia显卡才能达到最好效果！！！

一、 准备

  1. Windows项目提供者给出的编译工具是MSVS 2015, CUDA 8.0 and OpenCV 3.0，其实OpenCV只要是3.0以上的都可以，我这里使用的是OpenCV3.2。需要注意的是OpenCV的路径C:\opencv_3.0\opencv\build\include & C:\opencv_3.0\opencv\build\x64\vc14\lib。
  2. 将OpenCV的两个dll：opencv_world320.dll和opencv_ffmpeg320_64.dll放到C:\opencv_3.0\opencv\build\x64\vc14\bin目录下，之后生成的darknet.exe也需要这两个dll，没有这两个文件darknet.exe就不能正常运行。
  3. 下载CUDNN, 6.0以上版本即可。
    • 添加CUDNN到环境变量
    • 使用VS2015打开项目，右键选择属性 -> C/C++ -> 预处理器 -> 预处理器定义，并且在最开始添加CUDNN;
  4. 然后启动VS2015，打开darknet项目，设置为解决方案为Release，平台为x64。右键生成项目即可。
